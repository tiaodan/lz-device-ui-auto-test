## 第三步：理解页面对象（5分钟）

以 `LoginPage` 为例，所有页面对象结构一样：

```python
class LoginPage:
    def __init__(self, page: Page, language: str = "zh_CN"):
        self.page = page
        # 1. 定位器：元素在哪
        self.username_input = page.locator(".username")
        self.password_input = page.locator(".password")
        self.login_button = page.locator(".login-btn")

    # 2. 操作方法：能做什么
    def fill_credentials(self, username, password):
        self.username_input.fill(username)
        self.password_input.fill(password)

    def click_login(self):
        self.login_button.click()

    def drag_slider(self):
        # 滑块处理逻辑
        pass

    # 3. 验证方法：结果应该怎样
    def verify_elements_visible(self):
        expect(self.username_input).to_be_visible()
        expect(self.login_button).to_be_visible()
```

### 在测试中使用

```python
def test_login(login_page):
    # 操作
    login_page.fill_credentials("root", "password")
    login_page.drag_slider()
    login_page.click_login()

    # 验证
    login_page.verify_elements_visible()
```

## 编码套路总结

### 新增一个页面

```
1. 复制 page_template.py → pages/xxx_page.py
2. 改定位器（从实际页面找 CSS 选择器）
3. 添加操作方法
4. 添加验证方法
```

### 新增一个测试

```
1. 在对应目录创建 test_xxx.py
2. 写测试函数，参数写需要的 fixture
3. 调用页面对象操作
4. 用断言验证结果
```

### 新增一个 API 查询

```
1. 在 graphql_client.py 添加方法
2. 调用 self._execute(query_string)
3. 返回 result
```
