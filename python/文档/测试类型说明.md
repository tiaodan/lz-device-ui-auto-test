# 测试类型说明

## 三种测试类型

### 1. 断言测试 (assertion_test)

验证 UI 元素的存在性、可见性、属性、文本等。

**技术**：Playwright `expect` 断言

**测试内容**：
- 元素可见性 (`to_be_visible`)
- 元素可用性 (`to_be_enabled`)
- 元素数量 (`to_have_count`)
- 属性值 (`to_have_attribute`)
- 文本内容 (`to_have_text`, `to_contain_text`)
- 输入框值 (`to_have_value`)
- URL (`to_have_url`)
- 标题 (`to_have_title`)

**示例**：
```python
@pytest.mark.assertion
def test_login_page_elements(page, login_page, assertion_test, test_logger):
    assertion_test.assert_visible(login_page.username_input, "用户名输入框")
    assertion_test.assert_attribute(login_page.username_input, "placeholder", "Username")
```

**基类方法** (assertion_test/base.py)：
| 方法 | 说明 |
|-----|------|
| `assert_visible()` | 断言可见 |
| `assert_hidden()` | 断言隐藏 |
| `assert_enabled()` | 断言可用 |
| `assert_disabled()` | 断言禁用 |
| `assert_text()` | 断言文本精确匹配 |
| `assert_text_contains()` | 断言文本包含 |
| `assert_attribute()` | 断言属性值 |
| `assert_input_value()` | 断言输入框值 |
| `assert_url()` | 断言 URL |

### 2. 视觉测试 (visual_test)

通过截图对比检测 UI 变化，实现视觉回归测试。

**技术**：Pillow 图片像素对比

**测试内容**：
- 页面整体截图对比
- 元素截图对比
- 区域截图对比
- 差异百分比计算
- 差异图生成

**示例**：
```python
@pytest.mark.visual
def test_login_ok_visual(page, login_page, visual_test, test_logger):
    visual_test.take_screenshot("login_page_initial")
    if visual_test.has_baseline("login_page_initial"):
        result = visual_test.compare_images("login_page_initial")
        if not result["match"]:
            pytest.fail(f"差异 {result['diff_percent']}%")
```

**基类方法** (visual_test/base.py)：
| 方法 | 说明 |
|-----|------|
| `take_screenshot()` | 全页面截图 |
| `take_element_screenshot()` | 元素截图 |
| `take_region_screenshot()` | 区域截图 |
| `has_baseline()` | 检查基准图存在 |
| `compare_images()` | 图片对比 |
| `compare_with_baseline()` | 与指定基准图对比 |
| `update_baseline()` | 更新基准图 |
| `visual_test()` | 完整测试流程 |

**截图目录**：
```
screenshots/
├── baseline/     # 基准图（期望状态）
├── current/      # 当前截图
├── diff/         # 差异图（红色标记）
└── failures/     # pytest失败自动截图
```

**创建基准图**：
```bash
# 首次运行会自动创建（如果 auto_update_baseline=true）
# 或手动创建：
pytest --headed  # 手动截图保存到 baseline/
```

### 3. API测试 (api_test)

验证 GraphQL 接口的正确性。

**技术**：requests + GraphQL query/mutation

**测试内容**：
- 登录接口验证
- 查询接口验证（无人机、黑名单、白名单）
- 操作接口验证（打击、添加黑名单）
- 错误响应验证
- UI与API数据一致性验证

**示例**：
```python
@pytest.mark.api
def test_login_ok(api_test, test_logger, config):
    result = api_test.login(config.username, config.password)
    if result.get("token"):
        test_logger.pass_("返回包含 token")
    else:
        pytest.fail("登录失败")
```

**基类方法** (api_test/base.py)：
| 方法 | 说明 |
|-----|------|
| `login()` | 登录获取Token |
| `query_drones()` | 查询无人机 |
| `query_blacklist()` | 查询黑名单 |
| `query_whitelist()` | 查询白名单 |
| `add_blacklist()` | 添加黑名单 |
| `delete_blacklist()` | 删除黑名单 |
| `attack()` | 精准打击 |
| `assert_response_success()` | 断言响应成功 |
| `verify_ui_data_consistency()` | UI与API数据一致性 |

## Mock测试 (mock_test_this_testtool)

用于验证自动化测试程序本身的有效性。

**目的**：通过模拟UI异常，验证断言测试能否检测到问题。

**测试内容**：
- 模拟元素缺失
- 模拟元素隐藏
- 模拟元素重复
- 模拟元素位置错误
- 模拟元素禁用
- 模拟白屏

**检测结果**：
- 12个异常场景，检测率75% (9/12)
- 盲点：禁用元素、位置错误

**文件**：
- `mock_test_this_testtool/anomaly_simulator.py` - UI异常模拟器
- `mock_test_this_testtool/verify_assertion.py` - 验证断言有效性
