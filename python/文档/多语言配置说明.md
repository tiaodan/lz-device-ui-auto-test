# 多语言配置说明

## 概述

测试框架支持多语言。页面语言由 `localStorage.locale` / `sessionStorage.locale` / `document.lanType` 控制，测试框架通过 `add_init_script` 在页面加载前注入语言设置，确保页面按配置显示对应语言。

## 页面语言优先级

```
sessionStorage.locale -> localStorage.locale -> document.lanType(后台配置) -> navigator.language(浏览器) -> en_US
```

**关键点：** 页面读取的 storage key 是 `locale`，不是 `language`。写错 key 会导致注入不生效，页面仍显示默认语言。

## 配置方式

### 1. 全局配置指定语言

```yaml
# config/test_config.yaml
language: "zh_CN"
```

### 2. 支持的语言

| 配置值 | 语言 |
|--------|------|
| zh_CN | 简体中文 |
| en_US | English |
| fr_FR | Français |
| ru_RU | Русский |
| it_IT | Italiano |
| es_ES | Español |
| ko_KR | 한국어 |
| ar_EG | العربية |
| th_TH | ภาษาไทย |

### 3. 语言注入原理

conftest.py 在每个浏览器 context 创建时注入：

```python
context.add_init_script(f"""
    localStorage.setItem('locale', '{language}');
    sessionStorage.setItem('locale', '{language}');
    Object.defineProperty(document, 'lanType', {{ value: '{language}', writable: true }});
""")
```

这样页面 JS 执行前，语言已设置好，页面会按指定语言渲染。

## 配置文件结构

### 按语言拆分期望值

每个模块的期望值按语言独立文件：

```
config/login/
├── login_config.yaml    # 定位器 + 测试数据（不包含期望值）
├── zh_CN.yaml           # 中文期望值
├── en_US.yaml           # 英文期望值
├── fr_FR.yaml           # 法语期望值（新增语言只需加文件）
└── ...
```

### login_config.yaml（定位器 + 测试数据）

```yaml
locators:
  username_input: "#username"
  password_input: "#password"
  login_button: ".lBtn"
  ...

test_data:
  valid:
    username: "root"
    password: "lzno1root"
  invalid:
    username_not_exist: "root1"
    ...
```

### zh_CN.yaml（中文期望值）

```yaml
username_placeholder: "用户名"
password_placeholder: "密码"
login_button_text: "登录"
slider_text_contains: "滑块"
```

### en_US.yaml（英文期望值）

```yaml
username_placeholder: "Username"
password_placeholder: "Password"
login_button_text: "Sign in"
slider_text_contains: "slide"
```

## 测试代码使用

```python
def test_login_ok(login_page, assertion_test, test_logger, config, language):
    # 获取当前语言的期望值
    exp = login_page.get_expectations(language)

    # 用期望值断言，不再硬编码字符串
    assertion_test.assert_attribute(
        login_page.username_input,
        "placeholder",
        exp["username_placeholder"]
    )
    assertion_test.assert_text_contains(
        login_page.login_button,
        exp["login_button_text"]
    )
```

## 新增语言步骤

1. 在 `config/{module}/` 下新建语言文件，如 `ko_KR.yaml`
2. 填入该语言的期望值
3. 修改 `test_config.yaml`：`language: "ko_KR"`
4. 运行测试

无需修改任何 Python 代码。
