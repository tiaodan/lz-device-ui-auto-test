# 配置化说明

## 概述

定位器、断言期望值、测试数据已从代码中抽离到 YAML 配置文件，按模块分离。

## 配置目录结构

```
config/
├── test_config.yaml           # 全局配置（浏览器、日志、API地址等）
├── module_config_loader.py    # 模块配置加载器
└── login/
    └── login_config.yaml      # 登录模块配置
```

## login_config.yaml 结构

```yaml
# ==================== 定位器 ====================
locators:
  username_input: "#username"
  password_input: "#password"
  login_button: ".lBtn"
  slider_container: ".drag"
  slider_btn: ".drag .btn"
  slider_progress: ".drag .progress"
  slider_text: ".drag .text"

# ==================== 断言期望值 ====================
expectations:
  username_placeholder: "Username"
  password_placeholder: "Password"
  login_button_text: "Sign in"
  slider_text_contains: "slide"

# ==================== 测试数据 ====================
test_data:
  valid:
    username: "root"
    password: "lzno1root"
  invalid:
    username_not_exist: "root1"
    wrong_password: "lz"
    empty_username: ""
    empty_password: ""
```

## 使用方式

### 页面对象中使用配置

```python
from config.module_config_loader import get_module_config

class LoginPage:
    def __init__(self, page):
        self._config = get_module_config("login")
        locators = self._config["locators"]
        
        # 从配置初始化定位器
        self.username_input = page.locator(locators["username_input"])
        self.password_input = page.locator(locators["password_input"])
```

### 测试中使用配置数据

```python
@pytest.fixture
def login_config():
    return get_module_config("login")

def test_login_ok(login_page, login_config):
    test_data = login_config["test_data"]["valid"]
    login_page.fill_credentials(test_data["username"], test_data["password"])
```

### 获取配置的方法

```python
# 获取整个模块配置
config = get_module_config("login")

# 获取单个定位器
locator = get_locator("login", "username_input")  # 返回 "#username"

# 获取单个期望值
expectation = get_expectation("login", "username_placeholder")  # 返回 "Username"

# 获取测试数据
data = get_test_data("login", "valid")  # 返回 {"username": "root", "password": "lzno1root"}
```

## 好处

1. **维护性**：页面改版只需修改 YAML，无需改代码
2. **可读性**：配置文件一目了然
3. **扩展性**：新增模块只需新增配置文件（如 `config/drone/drone_config.yaml`）
4. **团队协作**：非程序员也能修改定位器
